# Спецификация: Сохранение состояния вкладок

## Контекст

При закрытии Electron-приложения eTty все вкладки теряются. Пользователь хочет сохранять состояние вкладок (открытые директории) между сессиями и восстанавливать их при запуске приложения.

## Требования

### REQ-1: Сохранение состояния при закрытии
- При закрытии приложения (событие `before-quit` или `will-quit`) автоматически сохранять состояние вкладок
- Сохраняемые данные для каждой вкладки:
  - `rootPath` — текущая рабочая директория (из OSC 7)
  - `isActive` — флаг активной вкладки (только одна)
- Файл хранится в `app.getPath('userData')` — `tabs-state.json`

### REQ-2: Структура файла состояния
```json
{
  "version": 1,
  "tabs": [
    { "rootPath": "/Users/me/project", "isActive": true },
    { "rootPath": "/Users/me", "isActive": false }
  ],
  "savedAt": "2026-04-16T10:30:00.000Z"
}
```

### REQ-3: Диалог восстановления при запуске
- При запуске приложения проверять наличие сохранённого состояния
- Если вкладки есть и приложение запускается нормально (не via dock click с документом):
  - Показывать нативный диалог: "Восстановить N вкладок?" с кнопками [Восстановить] [Не восстанавливать]
- Кнопка по умолчанию: **Не восстанавливать** (безопасность)

### REQ-4: Логика восстановления
- При нажатии "Восстановить":
  - Закрыть автоматически созданную первую вкладку
  - Создать вкладки в порядке сохранения
  - Активировать вкладку с `isActive: true`
- При нажатии "Не восстанавливать" или закрытии диалога:
  - Удалить файл состояния
  - Продолжить обычную работу (одна вкладка)

### REQ-5: Обработка отсутствия shell
- Если сохранённая директория не существует или shell не может запуститься в ней:
  - Использовать домашнюю директорию как fallback
  - Логировать предупреждение

### REQ-6: Кнопка "Восстановить вкладки"
- Добавить пункт в меню приложения: **Вкладки → Восстановить вкладки**
- Активировать только если есть сохранённое состояние
- При отсутствии файла — кнопка disabled

## Ограничения

- **Без scrollback**: сохраняется только директория, не история терминала
- **Без вкладок без директории**: если rootPath пустой — вкладка не сохраняется
- **Диалог один раз**: не показывать повторно в той же сессии
- **Single window**: нет поддержки нескольких окон (не в скоупе)

## Макеты и референсы

Не применимо — нативный диалог Electron (MessageBox).

## Кодстайл и конвенции

- Использовать `electron-store` для записи JSON (установить как зависимость)
- Путь к файлу: `path.join(app.getPath('userData'), 'tabs-state.json')`
- Обработка ошибок: логировать в `log.error()`, не падать
- Асинхронность: все операции с файлами — async/await

## Переиспользуемые решения

| Компонент | Путь | Назначение |
|-----------|------|------------|
| TabBar | `src/renderer/tab-bar.js` | Управление вкладками, получение rootPath |
| PtyManager | `src/main/pty-manager.js` | Создание PTY с заданной директорией |
| electron-store | npm | Хранение JSON-состояния |

## Критерии приёмки

- [ ] При закрытии приложения создаётся `tabs-state.json` с массивом вкладок
- [ ] При запуске с существующим файлом появляется диалог "Восстановить N вкладок?"
- [ ] При выборе "Восстановить" открываются вкладки с сохранёнными директориями
- [ ] При выборе "Не восстанавливать" файл удаляется, открывается одна вкладка
- [ ] Кнопка "Восстановить вкладки" в меню активна при наличии файла
- [ ] При недоступности директории используется fallback на home
