# Спецификация: Git Panel

## Контекст

eTty — терминальное Electron-приложение без какой-либо git-интеграции. Фича добавляет:
1. **Нижний статус-бар** с кнопкой `± +N -N`, показывающей количество изменённых строк в git-репозитории активной вкладки.
2. **Git-панель** (полноэкранный оверлей, аналог страницы настроек) с просмотром diff по файлам, управлением ветками и базовыми git-операциями (commit, push, discard).

Git-контекст определяется по `rootPath` активной вкладки — simple-git ищет репозиторий вверх по дереву от этой директории.

## Требования

**Статус-бар:**
- REQ-1: В нижней части окна постоянно отображается полоса высотой 22px с кнопкой `± +{additions} -{deletions}`.
- REQ-2: Кнопка обновляется каждые 5 секунд (polling) на основе `rootPath` активной вкладки.
- REQ-3: При смене активной вкладки — немедленное внеплановое обновление кнопки.
- REQ-4: Если `rootPath` активной вкладки не является git-репозиторием — кнопка скрыта.

**Git-панель — открытие/закрытие:**
- REQ-5: Клик по кнопке статус-бара открывает git-панель (полноэкранный оверлей).
- REQ-6: Панель закрывается кнопкой `✕` или клавишей `Esc`.
- REQ-7: При открытии панель немедленно запрашивает актуальный статус и список веток.

**Git-панель — шапка и ветки:**
- REQ-8: В шапке отображается `<select>` с текущей веткой и списком всех локальных веток.
- REQ-9: Смена значения в `<select>` выполняет `git checkout <branch>`.
- REQ-10: Кнопка `[+ New]` открывает инлайн-инпут; Enter создаёт новую ветку и переключается на неё.
- REQ-11: Кнопка `[🗑 Delete]` удаляет текущую ветку; заблокирована если ветка одна или называется `main`/`master`.
- REQ-12: Ошибки операций с ветками отображаются красной строкой под шапкой на 4 секунды.

**Git-панель — список файлов и diff:**
- REQ-13: Список изменённых файлов отображается с путём, `+N` (зелёный) и `-N` (красный).
- REQ-14: Клик по файлу разворачивает инлайн-дифф, повторный клик сворачивает.
- REQ-15: Одновременно развёрнут только один файл.
- REQ-16: Diff рендерится вручную (без сторонних библиотек): строки `+` — зелёный фон, `-` — красный фон, hunk-заголовки `@@` — `var(--muted)` курсивом, моноширинный шрифт.

**Git-панель — коммит, пуш, discard:**
- REQ-17: Блок действий содержит: поле для сообщения коммита, кнопки `[Commit]`, `[Push]`, `[Discard all]`.
- REQ-18: `[Commit]` активна только если поле не пустое и есть изменения; выполняет `git commit -am "<message>"`, после успеха очищает поле и обновляет список файлов.
- REQ-19: `[Push]` активна если есть remote; показывает спиннер (`...`) и блокируется на время выполнения.
- REQ-20: `[Discard all]` требует инлайн-подтверждения `«Сбросить все изменения? [Да] [Нет]»`; выполняет `git checkout -- . && git clean -fd`.
- REQ-21: Во время любой операции все кнопки блока действий задисейблены.
- REQ-22: Успешный push показывает зелёную строку «Pushed successfully» на 2 секунды.
- REQ-23: Ошибки операций (commit, push, discard) отображаются красной строкой под шапкой на 4 секунды.

## Ограничения

- Staging по отдельным файлам или hunks — **не входит в скоуп** (только `git commit -am`).
- Работа с remote (fetch, pull, merge, rebase) — **не входит в скоуп** (только push).
- Удаление remote-веток — **не входит в скоуп** (только локальные).
- Работа с несколькими репозиториями одновременно — **не входит в скоуп**.
- Визуализация git log / history — **не входит в скоуп**.

## Кодстайл и конвенции

- Vanilla JavaScript, ES6 модули — никаких фреймворков.
- Классы с явным конструктором и публичными методами (паттерн SettingsPage, FileTree, TabBar).
- CSS-переменные для всех цветов: `var(--bg)`, `var(--surface)`, `var(--border)`, `var(--text)`, `var(--accent)`, `var(--green)`, `var(--red)`, `var(--muted)`.
- IPC-каналы в preload регистрируются в `contextBridge.exposeInMainWorld('api', {...})` — аналогично существующим `fs*`, `pty*`, `settings*`.
- Обработка ошибок: все git-операции оборачиваются в try/catch в `git-service.js`; IPC возвращает `{ ok: true }` или `{ error: string }`.
- Нет сторонних библиотек для рендера диффа — только встроенный парсинг unified diff формата.

- `src/renderer/index.js` — место инициализации StatusBar и GitPanel, подписка на смену активной вкладки.

## Критерии приёмки

- [ ] Статус-бар виден внизу окна, кнопка обновляется при смене вкладки и раз в 5 сек.
- [ ] Кнопка скрывается если `rootPath` не является git-репозиторием.
- [ ] Клик по кнопке открывает git-панель, `Esc` и `✕` закрывают.
- [ ] В шапке отображается текущая ветка, переключение через `<select>` работает.
- [ ] Создание новой ветки через `[+ New]` + инпут работает.
- [ ] Удаление ветки заблокировано для `main`/`master` и единственной ветки.
- [ ] Список изменённых файлов отображается с корректной статистикой `+N -N`.
- [ ] Клик по файлу раскрывает diff, повторный клик сворачивает; одновременно один файл.
- [ ] Строки diff корректно окрашены (зелёный/красный/muted для hunks).
- [ ] `[Commit]` недоступна при пустом поле; после успешного коммита список обновляется.
- [ ] `[Push]` показывает спиннер во время выполнения.
- [ ] `[Discard all]` требует подтверждения перед выполнением.
- [ ] Все ошибки отображаются без падения приложения.
- [ ] UI соответствует визуальному стилю eTty (CSS-переменные тем, шрифты, отступы).
