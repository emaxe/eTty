# Стартовый промпт для реализации фичи «Git Panel»

Скопируй целиком и вставь в новую сессию.

---

Реализуй фичу **Git Panel** по подготовленной документации.

## Документация

- Спецификация: `docs/features/git-panel/spec.md` — прочитай перед началом
- План: `docs/features/git-panel/plan.md` — следуй порядку задач, секциям «Стратегия выполнения» и «Ревью после каждого шага»
- Чеклист: `docs/features/git-panel/checklist.md` — отмечай прогресс

## Инструкции

1. Прочитай `spec.md`, `plan.md` и `checklist.md` целиком.
2. Создай/переключись на ветку `feature/git-panel` (`git checkout -b feature/git-panel` или `git checkout feature/git-panel`).
   **Базовая ветка для merge после завершения: `main`.** Не меняй без причины.
3. Отметь в checklist.md пункты «Подготовка» как выполненные.
4. Выполняй задачи согласно стратегии выполнения из `plan.md`:
   - Блок 1 (задачи #1–3) и Блок 2 (задачи #4–5) — строго последовательно.
   - Задачи #6 и #7 (Блок 3) — можно поручить параллельным субагентам одновременно: они работают с разными файлами и не зависят друг от друга.
   - Задача #8 (Блок 4) — только после завершения обеих задач #6 и #7.
5. После **каждой задачи**: сверься с `plan.md` и `spec.md`, убедись что не нарушаешь параллельные задачи.
6. Если задачу делал субагент — проверь результат перед следующим шагом.
7. После каждой задачи отмечай её в `checklist.md` (`[x]`).
8. Запускай проверку, указанную для задачи в плане.
9. Коммить после каждого блока (`git commit -m "feat: ..."`) — не после каждой задачи.
10. По завершении — отметь «Финализация» в `checklist.md` и обнови статус в `README.md` на `Done`.
11. Если что-то непонятно — спроси, не додумывай.

## Ключевые детали архитектуры

- Стек: **vanilla JS, ES6 модули**, Electron 33 — никаких фреймворков.
- `git-service.js` — экспортирует `registerGitHandlers(ipcMain)`, вызывается в `src/main/index.js`.
- `GitPanel` — аналог `SettingsPage` (overlay с `hidden` классом, методы `show(rootPath)`/`hide()`).
- `StatusBar` — polling каждые 5с, `updateNow()` при смене вкладки.
- Diff рендерится вручную (парсинг unified diff, без сторонних библиотек).
- Все цвета через CSS-переменные: `var(--green)`, `var(--red)`, `var(--muted)` и т.д.
- IPC-каналы: `git:get-status`, `git:get-diff`, `git:get-branches`, `git:checkout`, `git:create-branch`, `git:delete-branch`, `git:commit`, `git:push`, `git:discard`.
